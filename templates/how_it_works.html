Trust architecture
How CertifyMe keeps certificates safe
H
SHA-256 hashing

When a certificate is uploaded, a SHA-256 hash is generated from its content so even a tiny modification produces a completely different hash value.

V
Verifiable links

Each certificate is assigned a unique verification code and optional public link. Anyone with the code can check authenticity on the Verify page — no QR required.

✓
Tamper detection

On verification, the system recomputes the hash and compares with the original. Any mismatch means tampering.

Flow
Simple three-step process
1
Teacher uploads

Teacher uploads a PDF and fills student details. Metadata and a secure hash are stored.

2
Hash & HMAC

The hash is combined with a secret HMAC key only the platform knows.

3
Verify anytime

Anyone can verify instantly. If changed, it is flagged as invalid.
